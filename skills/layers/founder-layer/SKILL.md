# Founder Layer

This group covers the full business stack: fundraising, go-to-market, content, and customer-facing communication. When activated, consult the relevant sub-skill below and follow its workflow.

## Skill Map

### Fundraising & Investor Relations
- **investor-materials** — pitch decks, one-pagers, financial models, accelerator applications (Allan Gray, YC, etc.), use-of-funds tables, milestone plans. Use when creating or updating any investor-facing document.
- **investor-outreach** — cold emails, warm intro requests, follow-ups, post-meeting updates. Enforces proof-over-adjectives discipline and bans weak phrases. Use when writing any message to an investor or partner.
- **market-research** — TAM/SAM/SOM sizing, competitive analysis, investor diligence prep, technology scans. Use when you need data to back a claim or a decision.
- **competitive-landscape** — Porter's Five Forces, Blue Ocean positioning, competitor deep-dives. Use when mapping the competitive field or preparing a differentiation narrative.

### Pricing & Revenue
- **pricing-strategy** — pricing tiers, freemium vs paid, value metric, willingness to pay, Van Westendorp. Use when deciding what to charge or how to structure plans.

### Content & Communication
- **brand-voice** — builds a reusable voice profile from real source material. Run this first before writing anything external; produces a VOICE PROFILE block that other skills consume.
- **copywriting** — landing page copy, hero sections, CTAs, value propositions, taglines. Use when writing website or marketing text.
- **article-writing** — blog posts, LinkedIn articles, technical essays, newsletter issues. Use when producing long-form content.
- **launch-strategy** — Product Hunt, feature announcements, GTM plans, launch checklists, beta/waitlist strategy. Use when shipping something publicly.

## How to Use

1. Identify which sub-skill applies to the request.
2. Read that sub-skill's SKILL.md at `/mnt/skills/user/<skill-name>/SKILL.md`.
3. Follow its workflow exactly — each has a quality gate before delivering output.
4. For investor documents: always establish a single source of truth for metrics before drafting anything.
5. For outreach: always personalize — generic drafts are explicitly banned.

## Golden Rules Across All Sub-Skills

- Numbers must be sourced or labeled as estimates.
- Every investor document must be internally consistent — if the deck says one thing and the model says another, stop and resolve before writing.
- Outreach language: proof beats adjectives. Never send copy that could go to any recipient.
- Voice: run brand-voice first when the user's voice matters, then hand the VOICE PROFILE to other skills.
